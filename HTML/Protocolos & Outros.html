SSL vs TLS

SSL (Secure Sockets Layer) e TLS (Transport Layer Security) são ambos protocolos criptográficos projetados para fornecer comunicação segura em uma rede de computadores. O TLS é basicamente uma versão atualizada e mais segura do SSL. Aqui estão alguns pontos-chave de comparação:

Foram criados para a comunicação FIM a FIM, usando criptografia simétrica e assimétrica.

Visam promover a segurança em comuncações na internet, buscando gatantir a:

  • Confidencialidade (...se os dados estão protegidos);
  • Autenticidade (...de onde vieram, qual a autoria, garantia do não repudio);
  • Integridade (...se os dados não foram alterados).

1. Desenvolvimento:

O TLS é o sucessor do SSL e foi introduzido para abordar vulnerabilidades e deficiências encontradas no SSL.

2. Versões:

O SSL possui várias versões, incluindo SSL 2.0 e SSL 3.0.

O TLS tem múltiplas versões, incluindo TLS 1.0, TLS 1.1, TLS 1.2, TLS 1.3 e potencialmente mais no futuro.

3. Compatibilidade Retroativa:

O TLS é projetado para ser retrocompatível com o SSL. Isso significa que um servidor habilitado para TLS pode suportar conexões de clientes usando SSL.

4. Melhorias de Segurança:

O TLS inclui várias melhorias de segurança em relação ao SSL. Por exemplo, o TLS possui algoritmos de criptografia mais fortes, métodos de troca de chaves mais seguros e verificações de integridade aprimoradas.

5. Vulnerabilidades:

O SSL foi encontrado com vulnerabilidades, especialmente em suas versões mais antigas (SSL 2.0 e SSL 3.0).

O TLS é projetado para abordar as fraquezas de segurança do SSL, e novas versões do TLS buscam constantemente aprimorar a segurança.

6. Últimas Versões:

A última versão do SSL é a SSL 3.0, mas não é recomendada para uso devido a vulnerabilidades conhecidas.

Até a minha última atualização de conhecimento em janeiro de 2022, o TLS 1.3 é a versão mais recente e segura. O TLS 1.3 inclui melhorias significativas em termos de desempenho e segurança.

7. Perfect Forward Secrecy (PFS):

O TLS enfatiza mais fortemente a Perfect Forward Secrecy (PFS) por padrão, garantindo que, mesmo se uma chave secreta de longo prazo for comprometida, as comunicações passadas permaneçam seguras.

8. Adoção pela Indústria:

O SSL é obsoleto e considerado inseguro, sendo fortemente desencorajado seu uso.

O TLS é amplamente adotado e considerado o padrão para garantir comunicações pela Internet.

AMPQ vs MQTT

AMPQ e MQTT - São dois protocolos de mensagens muito populares, cada um com suas próprias vantagens e desvantagens.

AMPQ é um protocolo de mensagens de propósito geral projetado para ser robusto e escalável. Ele oferece uma ampla gama de recursos, incluindo:

  • QoS (Quality of Service) de 0 a 2, garantindo que as mensagens sejam entregues com segurança e confiabilidade.
  • Transações, permitindo que um grupo de mensagens seja tratado como uma única unidade.
  • Roteamento, permitindo que as mensagens sejam enviadas para destinos específicos.
  • Armazenamento de mensagens, permitindo que as mensagens sejam armazenadas para posterior processamento.

MQTT é um protocolo de mensagens leve projetado para dispositivos com recursos limitados. Ele é simples e fácil de implementar, e oferece os seguintes recursos:

  • QoS de 0 a 2, garantindo que as mensagens sejam entregues com segurança e confiabilidade.
  • Publicação/assinatura, permitindo que os dispositivos publiquem mensagens que possam ser consumidas por outros dispositivos.

SAML vs OIDC

SAML e OIDC são protocolos para configuração para conexões de autenticação para aplicativos.

Como fornecedor de soluções de gerenciamento de identidade e acesso (IAM) , oferecemos aos nossos usuários diversas opções quando eles precisam configurar conexões de autenticação para aplicativos. Dois dos principais protocolos de autenticação são:

  • Linguagem de marcação de acesso de segurança (SAML)
  • Conexão OpenID (OIDC)

A maioria dos aplicativos oferece suporte apenas a uma dessas opções, mas ocasionalmente você pode ter que escolher entre SAML e OIDC . Portanto, pode ser útil saber qual é a diferença para que você possa escolher qual funcionará melhor para sua organização.

Como o SAML e o OIDC são semelhantes?

Tanto SAML quanto OIDC são considerados protocolos de identidade. Eles fornecem um meio pelo qual os usuários podem ser autenticados e as informações do usuário podem ser transmitidas com segurança entre o sistema que está fazendo a autenticação, também conhecido como Provedor de Identidade (IdP) e o serviço ou aplicativo que o usuário está tentando acessar. Uma das principais etapas para habilitar esta forma de Single Sign-On (SSO) é que uma relação de confiança deve ser estabelecida entre o IdP e o aplicativo. SAML e OIDC são protocolos importantes usados ​​em qualquer solução de SSO porque o objetivo de uma solução de SSO é que os usuários precisarão se autenticar apenas uma vez com o IdP e então poderão acessar qualquer um dos aplicativos que foram configurados para confiar no IdP.

O fluxo básico de login para ambos é o mesmo.

  1. Um usuário efetua login no provedor de identidade.
  2. Eles selecionam qual aplicativo desejam acessar.
  3. As informações do usuário são passadas do IdP para o navegador do usuário.
  4. Suas informações são então repassadas para o aplicativo.
  5. O aplicativo confirma que eles estão autorizados a acessar os recursos.
  6. O usuário tem permissão para entrar no aplicativo.

O fluxo de login também pode ser assim.

  1. Um usuário tenta fazer login diretamente no aplicativo.
  2. O aplicativo redireciona a solicitação de login através do navegador do usuário para o IdP.
  3. O usuário efetua login no IdP ou é confirmado que já está conectado ao IdP
  4. O IdP confirma que o usuário tem acesso ao aplicativo que enviou a solicitação.
  5. As informações do usuário são passadas do IdP para o navegador do usuário.
  6. Suas informações são então repassadas para o aplicativo.
  7. O aplicativo confirma que eles estão autorizados a acessar os recursos.
  8. O usuário tem permissão para entrar no aplicativo.

Qual a diferença entre SAML e OIDC?

SAML

SAML é mais antigo que OIDC. O SAML entrou em cena pela primeira vez em 2005. Ele transmite dados como nomes de usuário, nomes, sobrenomes dos usuários, etc.

SAML refere-se ao aplicativo como Provedor de Serv

Na verdade, o primeiro fluxo descrito acima é conhecido como SSO iniciado pelo provedor de identidade (iniciado pelo IdP).

O fluxo que começa com o usuário tentando fazer login diretamente no aplicativo ou SP primeiro é chamado de SSO iniciado pelo provedor de serviços (iniciado pelo SP).

SAML ainda é um dos protocolos SSO mais populares em uso atualmente.

OIDC

OIDC é baseado no protocolo OAuth 2.0. Enquanto o OAuth 2.0 é usado para configurar dois aplicativos, como dois sites, que podem confiar um no outro e enviar e receber dados, o OIDC funciona no nível individual ou do usuário.

Em comparação com SAML, os fluxos de login do OIDC funcionam da mesma maneira. Mas existem três diferenças principais:

  1. SAML transmite dados do usuário em formato XML. OIDC transmite dados do usuário no formato JSON.
  2. SAML chama os dados do usuário e envia uma asserção SAML. OIDC chama as declarações de dados.
  3. SAML chama o aplicativo ou sistema que o usuário está tentando acessar para o provedor de serviços. OIDC chama isso de Parte Confiante.

Portanto, o fluxo geral parece o mesmo, apenas os rótulos são diferentes.

OpenID Connect está ganhando popularidade. É muito mais simples de implementar do que SAML e facilmente acessível por meio de APIs porque funciona com endpoints de API RESTful. Isso também significa que funciona muito melhor com aplicativos móveis.

Ao configurar o SSO para um aplicativo por meio de um IdP como o OneLogin, você nem sempre terá a opção entre SAML ou OIDC . Suas escolhas são baseadas no que os desenvolvedores de aplicativos escolhem oferecer suporte. Mas se você tiver escolha, é importante entender qual é a diferença entre os dois e qual deles tem maior probabilidade de ser suportado por mais tempo. Neste momento, como os desenvolvedores consideram o OIDC muito mais fácil de trabalhar e por ser mais flexível, o OIDC parece que será o vencedor.

SAML (Security Assertion Markup Language) é uma linguagem de marcação baseada em XML usada para autenticação via federação de identidade. Em outras palavras, ele permite que um usuário se autentique em um sistema e obtenha acesso a outro sistema, fornecendo prova de sua autenticação.

O SAML funciona estabelecendo uma conexão segura entre três partes:

  • O usuário: O usuário é a pessoa que está tentando se autenticar.
  • O provedor de identidade (IdP): O IdP é a entidade que gerencia as credenciais de identidade do usuário.
  • O provedor de serviços (SP): O SP é a entidade que fornece o serviço ao qual o usuário está tentando acessar.

O processo de autenticação SAML ocorre da seguinte forma:

  • O usuário inicia o processo de autenticação ao fornecer suas credenciais ao IdP.
  • O IdP autentica o usuário e gera uma afirmação de autenticação.
  • O IdP envia a afirmação de autenticação ao SP.
  • O SP verifica a afirmação de autenticação e concede acesso ao usuário ao serviço.

As afirmações de autenticação SAML contêm informações sobre a identidade do usuário, como o nome de usuário, o endereço de e-mail e o grupo do usuário. O SP pode usar essas informações para conceder acesso ao usuário ao serviço.

O SAML é um protocolo de autenticação seguro e flexível que pode ser usado em uma variedade de cenários. É frequentemente usado para implementar login único (SSO), que permite que os usuários se autentiquem uma vez e obtenham acesso a vários serviços.

Aqui estão alguns exemplos de como o SAML é usado:

  • Login único: O SAML pode ser usado para implementar login único em uma organização. Isso permite que os usuários se autentiquem uma vez com o IdP da organização e obtenham acesso a todos os serviços da organização.
  • Federação de identidade: O SAML pode ser usado para federar identidades entre organizações. Isso permite que os usuários se autentiquem com um IdP de uma organização e obtenham acesso aos serviços de outra organização.
  • Autenticação de dois fatores: O SAML pode ser usado para implementar autenticação de dois fatores. Isso adiciona uma camada extra de segurança à autenticação, exigindo que os usuários forneçam um código de verificação além de suas credenciais de login.

O SAML é um protocolo de autenticação importante que pode ser usado para melhorar a segurança e a eficiência das comunicações.

OpenID Connect (OIDC) é um protocolo de autenticação de identidade que utiliza os mecanismos de autorização e autenticação do OAuth 2.0. O OIDC é projetado para aplicativos web e móveis e fornece uma maneira segura e fácil para os usuários se autenticarem em aplicativos e serviços.

O OIDC funciona da seguinte forma:

  1. O usuário visita um aplicativo que usa o OIDC.
  2. O aplicativo solicita ao usuário que se autentique com um provedor de identidade (IdP).
  3. O usuário é redirecionado para o IdP para fazer login.
  4. O IdP autentica o usuário e emite um token de ID.
  5. O usuário é redirecionado de volta para o aplicativo com o token de ID.

O token de ID é um token criptografado que contém informações sobre o usuário, como seu nome, e-mail e endereço IP. O aplicativo usa o token de ID para verificar a identidade do usuário e fornecer acesso ao aplicativo ou serviço.

O OIDC oferece uma série de benefícios, incluindo:

  • Segurança: O c usa criptografia para proteger as informações do usuário.
  • Facilidade de uso: O OIDC é fácil de implementar e usar.
  • Compatibilidade: O OIDC é compatível com muitos aplicativos e serviços.

O OIDC é um protocolo de autenticação de identidade popular que é usado por uma ampla gama de aplicativos e serviços. É uma escolha segura e fácil para aplicativos web e móveis.

Aqui estão alguns exemplos de como o OIDC é usado:

  • Aplicativos de mídia social: Os aplicativos de mídia social, como Facebook e Twitter, usam o OIDC para permitir que os usuários se autentiquem com suas contas.
  • Aplicativos corporativos: As empresas usam o OIDC para permitir que os funcionários se autentiquem em aplicativos corporativos.
  • Sites de comércio eletrônico: Os sites de comércio eletrônico usam o OIDC para permitir que os usuários se autentiquem para fazer compras.

O OIDC é uma tecnologia importante que está sendo cada vez mais adotada por aplicativos e serviços. É uma escolha segura e fácil para aplicativos web e móveis que precisam de autenticação de identidade.

Assinatura Digital vs Certificado Ditigal

Assinatura digital e Certifiado Digital são dois recursos que permitem reforçar a segurança da troca de informações e de negociações formalizadas em ambientes virtuais. Ambos usam a chave ASSIMÉTRICA.

Assinatura Digital é um tipo de formalização de documentos eletrônicos em que os dados do titular de um certificado digital são vinculados ao documento assinado por meio de criptografia complexa.

Principais informações

  1. Usa a chave PRIVADA para criptografar um hash do documento ou mensagem a ser assinado;
  2. Integridade - Garante a não alteração do documento;
  3. Autenticidade - Garante que foi assinada pelo remetente;
  4. Não repúdio - Garante que não pode negar a transação eletrôica.

Resumo: A chave PRIVADA criar a assinatura, a chave PÚBLICA verifica a assinatura.



Assinatura Eletrônica:

A assinatura eletrônica é um conjunto de dados, no formato eletrônico, que é anexado ou logicamente associado a um outro conjunto de dados (o conteúdo que se quer assinar), também no formato eletrônico, para lhe conferir autenticidade ou autoria e, dependendo do tipo de assinatura eletrônica aplicada, integridade do conteúdo assinado eletronicamente.

Obs: Tanto a Assinatura Digital quanto o Certificado Digital podem utilizar diferentes tipos de criptografia, como RSA, ECC ou DSA.

RSA - (Racional Security Algorithm) - "Algoritmo de Segurança Racional". É um algoritmo de criptografia assimétrica que é amplamente utilizado para garantir a segurança de dados digitais. Ele foi desenvolvido por Ron Rivest, Adi Shamir e Leonard Adleman em 1978 e é baseado na dificuldade de factorizar números primos grandes.

ECC - (Elliptic Curve Cryptography) - "Criptografia de Curva Elíptica". É um tipo de criptografia assimétrica que usa curvas elípticas para gerar chaves públicas e privadas.

DSA - (Digital Signature Algorithm) - "Algoritmo de Assinatura Digital". É um algoritmo de criptografia assimétrica que é usado para assinar digitalmente documentos e mensagens.

Criptografia Simétrica vs Criptografia Assimétrica

Simetrica e Assimétrica são dois tipos diferentes de criptografia que têm suas próprias vantagens e desvantagens. A criptografia simétrica é mais rápida e eficiente, mas é menos segura do que a criptografia assimétrica. A criptografia assimétrica é mais segura, mas é menos eficiente. A escolha do tipo de criptografia a ser usada depende da aplicação específica.

A principal diferença entre a criptografia simétricae a criptografia assimétrica é o número de chaves usadas. A criptografia simétrica usa uma única chave para criptografar e descriptografar os dados, enquanto a criptografia assimétrica usa duas chaves: uma chave pública e uma chave privada.

Criptografia simétrica - (SSL e TLS)

A criptografia simétrica é o tipo mais simples de criptografia. Ela usa uma única chave para criptografar e descriptografar os dados. Essa chave pode ser compartilhada entre as duas partes que desejam se comunicar.

A criptografia simétrica é rápida e eficiente, pois usa apenas uma chave. No entanto, ela é menos segura do que a criptografia assimétrica, pois a chave secreta pode ser comprometida se cair em mãos erradas.

Criptografia assimétrica - (SSH e HTTPS)

A criptografia assimétrica é mais complexa do que a criptografia simétrica, mas também é mais segura. Ela usa duas chaves: uma chave pública e uma chave privada. A chave pública pode ser divulgada para qualquer pessoa, enquanto a chave privada deve ser mantida em segredo pelo proprietário.

Para criptografar os dados, é usada a chave pública do destinatário. Para descriptografar os dados, é usada a chave privada do remetente.

A criptografia assimétrica é usada para garantir a autenticidade, a integridade e a confidencialidade dos dados. Ela é usada em aplicações como assinatura digital, comércio eletrônico e troca de chaves.

SSL (Secure Sockets Layer) é um protocolo de segurança que fornece comunicação criptografada entre dois computadores. É usado para proteger comunicações na Internet, como navegação na web, e-mail e banco online.

Uso do SSL:

  • Quando você visita um site seguro, o endereço do site começa com "https". Isso significa que a comunicação entre seu navegador e o servidor do site é criptografada usando o SSL.
  • Quando você envia um e-mail, o e-mail é criptografado usando o SSL se você estiver usando um provedor de e-mail seguro.
  • Quando você faz compras online, seus dados de pagamento são criptografados usando o SSL para proteger suas informações financeiras.
  • VPNs, VoIP e Transferência de arquivos também são alguns outros exemplos de como o SSL é usado.

TLS (Transport Layer Security) é um protocolo de segurança utilizado para proteger a comunicação na Internet, garantindo a confidencialidade e integridade dos dados transmitidos. Sucessor do SSL.

Uso do TLS:

  • Tráfego HTTPS: O TLS é usado para proteger todo o tráfego HTTPS na Internet. HTTPS é a versão segura do HTTP e é usada para proteger todos os tipos de tráfego da web, incluindo navegação na web, e-mail e banco online.
  • Outros aplicativos: O TLS também é usado para proteger outros aplicativos que exigem comunicação segura pela Internet, como VPNs, voz sobre IP (VoIP) e mensagens instantâneas.

Tanto o SSL e o TLS fornecem uma série de recursos de segurança, incluindo:

  • Confidencialidade: Eles criptografam toda a comunicação entre o cliente e o servidor, para que apenas as partes pretendidas possam lê-la.
  • Integridade: Eles garantem que a comunicação entre o cliente e o servidor não seja alterada durante o transporte.
  • Autenticação: Eles permitem que o cliente verifique a identidade do servidor, para que o cliente possa ter certeza de que está se comunicando com o servidor correto.

SSH (Secure Shell) é um protocolo de rede criptografado que permite a comunicação segura e autenticada entre dois sistemas, geralmente usado para acesso remoto a servidores.

Uso do SSH:

  • Login remoto: O SSH é uma maneira popular de fazer login remoto em um computador. Isso permite que os usuários acessem e gerenciem um computador de outro local.
  • Transferência de arquivos: O SSH também pode ser usado para transferir arquivos com segurança entre computadores. Esta é uma maneira segura de compartilhar arquivos pela Internet.
  • Encaminhamento de portas: O SSH pode ser usado para encaminhar portas entre computadores. Isso pode ser usado para fornecer acesso seguro a serviços que estão sendo executados em outro computador.
