Padrão State

O padrão State é um padrão de projeto de software que permite que você altere o comportamento de um objeto alterando seu estado.

Benefícios

  • Flexibilidade: O padrão State permite que você altere o comportamento de um objeto sem alterar seu código.:
  • Coesão: O padrão State pode ajudar a melhorar a coesão do seu código, separando o comportamento de um objeto de seu estado.:
  • Testabilidade: O padrão State pode tornar seu código mais testável, tornando mais fácil testar o comportamento de um objeto em diferentes estados.:

Exemplos de uso deste padrão:

  • Uso do padrão State no desenvolvimento de software e hardware
    • Objetivo
      • Modelar o comportamento de um objeto ao longo do tempo
      • Reduzir a complexidade do código
      • Facilitar a manutenção do código
    • Exemplo
      • Desenvolvimento de software
        • Implementar diferentes estados de máquina
        • Implementar diferentes modos de operação
        • Implementar diferentes perfis de usuário
      • Desenvolvimento de jogos
        • Implementar diferentes estados de um personagem de jogo
        • Implementar diferentes estados de um jogo
        • Implementar diferentes estados de um nível de jogo
      • Desenvolvimento de hardware
        • Implementar diferentes estados de um dispositivo
        • Implementar diferentes modos de operação de um dispositivo
