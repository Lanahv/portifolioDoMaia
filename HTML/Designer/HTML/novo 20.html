Padrão Command

O padrão Command é um padrão de projeto de software que permite que você encapsule uma solicitação em um objeto.

Benefícios

  • Flexibilidade: O padrão Command permite que você encapsule diferentes tipos de solicitações em objetos.
  • Manutenção: O padrão Command torna seu código mais fácil de manter, pois você pode alterar a ordem ou a configuração das solicitações sem afetar o código que usa os comandos.
  • Testabilidade: O padrão Command pode tornar seu código mais testável, permitindo que você teste os comandos independentemente do código que os usa.

Exemplos de uso deste padrão:

  • Uso dos padrões Builder e Command no desenvolvimento de jogos e hardware
    • Objetivo
      • Facilitar a criação e execução de ações
      • Melhorar a flexibilidade e a manutenibilidade do código
      • Facilitar a reutilização de código
    • Exemplo
      • Desenvolvimento de jogos
        • Criar personagens
        • Executar ações em personagens
      • Desenvolvimento de hardware
        • Criar dispositivos físicos
        • Executar operações em dispositivos físicos
