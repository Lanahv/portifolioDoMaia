Padrão Mediator

O padrão Mediator é um padrão de projeto de software que permite que você decouple a comunicação entre objetos.: O padrão define uma classe abstrata Mediator que fornece métodos para comunicar objetos. Classes concretas implementam a classe Mediator para fornecer comunicação entre objetos específicos.

Exemplos de benefício do padrão Mediator

  • Flexibilidade: O padrão Mediator permite que você altere a comunicação entre objetos sem alterar os objetos diretamente.:
  • Escopo de controle: O padrão Mediator permite que você controle a comunicação entre objetos.:
  • Reutilização: O padrão Mediator permite que você reutilize o código para comunicar objetos.:

Exemplos de uso deste padrão:

| Sistema de e-commerce | * Comunicação entre os diferentes componentes do sistema * Carrinho de compras envia mensagem a checkout para finalizar uma compra * Checkout envia mensagem a banco de dados para registrar a compra * Sistema de recomendação envia mensagem a carrinho de compras para sugerir produtos | Sistema de gerenciamento de recursos humanos | * Comunicação entre os diferentes departamentos da empresa * Departamento de RH envia mensagem ao departamento financeiro para solicitar o pagamento de um funcionário * Departamento de TI envia mensagem ao departamento de RH para solicitar informações sobre um funcionário * Departamento de vendas envia mensagem ao departamento de marketing para solicitar materiais promocionais
