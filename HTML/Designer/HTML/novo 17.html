Padrão Visitor

O padrão Visitor é um padrão de projeto de software que permite que você adicione novas funcionalidades a objetos sem alterar sua estrutura.

Benefícios

  • Flexibilidade: O padrão Visitor permite que você adicione novas funcionalidades a objetos sem alterar sua estrutura.
  • Coesão: O padrão Visitor pode ajudar a melhorar a coesão do seu código, separando a lógica de processamento de dados da lógica de apresentação.
  • Testabilidade: O padrão Visitor pode tornar seu código mais testável, permitindo que você teste as funcionalidades de visitantes diferentes sem precisar testar os objetos visitados.

Exemplos de uso deste padrão:

| Desenvolvimento de jogos | * Aplicar uma operação a uma coleção de objetos * Aplicar um efeito visual a uma coleção de objetos de jogo. * Atualizar a posição de uma coleção de objetos de jogo. * Verificar se uma coleção de objetos de jogo está colidindo. * Aplicar uma operação a uma hierarquia de objetos * Aplicar um efeito sonoro a uma coleção de objetos de jogo. * Aplicar uma física a uma coleção de objetos de jogo. * Aplicar uma operação a uma coleção de objetos heterogêneos * Aplicar uma regra de jogo a uma coleção de objetos de jogo. * Executar uma ação em uma coleção de objetos de jogo de diferentes tipos. | Desenvolvimento de hardware | * Aplicar uma operação a uma coleção de objetos * Testar uma coleção de componentes eletrônicos. * Calibrar uma coleção de sensores. * Atualizar o firmware de uma coleção de dispositivos. * Aplicar uma operação a uma hierarquia de objetos * Aplicar um tratamento térmico a uma coleção de peças mecânicas. * Executar um teste de estresse a uma coleção de componentes eletrônicos. * Aplicar uma operação a uma coleção de objetos heterogêneos * Aplicar uma regra de segurança a uma coleção de dispositivos. * Executar uma ação em uma coleção de componentes de hardware de diferentes tipos.
