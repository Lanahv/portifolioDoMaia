Padrão Decorator

O padrão Decorator é um padrão de projeto de software que permite que você adicione funcionalidades a objetos existentes sem modificar o código desses objetos.

Benefícios

  • Flexibilidade: O padrão Decorator permite que você adicione funcionalidades a objetos existentes de forma flexível. Isso pode ser útil para adicionar diferentes tipos de funcionalidades a um objeto.:
  • Extensibilidade: O padrão Decorator pode ser usado para estender um sistema sem modificar o código dos objetos existentes. Isso pode ser útil para adicionar novos recursos ou funcionalidades a um sistema.:
  • Manutenção: O padrão Decorator pode facilitar a manutenção de um sistema. Isso ocorre porque ele separa a funcionalidade do objeto da sua implementação.:

Exemplos de uso deste padrão:

  • Aplicativo web
    • Adicionar suporte a criptografia a um objeto de rede
    • Adicionar suporte a um idioma específico a um objeto de interface do usuário
    • Adicionar suporte a log a um objeto de rede
  • Sistema de e-commerce
    • Personalizar um objeto de interface do usuário
    • Adicionar suporte a diferentes métodos de pagamento a um objeto de pedido
    • Adicionar suporte a diferentes condições de frete a um objeto de pedido
  • Sistema de gerenciamento de recursos humanos
    • Encapsular a lógica de log em um objeto de rede
    • Encapsular a lógica de auditoria em um objeto de banco de dados
    • Encapsular a lógica de cache em um objeto de objeto
  • Outros exemplos
    • Adicionar suporte a diferentes tipos de mídia a um objeto de arquivo
    • Adicionar suporte a diferentes formatos de saída a um objeto de impressão
    • Adicionar suporte a diferentes níveis de segurança a um objeto de acesso a dados
